Overview

How Rating + Shipping Insights Works with the PlaceOrder mutation‚Äã

Users integrate with ShipperHQ‚Äôs rating API to provide shipping rate estimates based on contents of a virtual cart. This is done leveraging the rating API.

Once a user has finalized his choices, the rating (a quote) need to be converted into an order. Usually, multiple quotes are send by ShipperHQ (for example in the cart, when the user changes her/his adress, when the user select delivery options, etc.).

PlaceOrder's role is to convert one of these quotes into a full fledge order so that the complete quote information can be stored on ShipperHQ order database and later on retrieved via the Insight API.

Most users don't need to use the PlaceOrder mutation.

If you are using an eCommerce platform Supported by ShipperHQ (Magento/Adobe Commerce, BigCommerce, Shopify, SalesForce Commerce Cloud, WooCommerce, Zoey - complete list here): you do not need to use PlaceOrder. In these scenario, the order creation is done automatically by the platform using ShipperHQ rates and ShipperHQ capture this information automatically so that you can use directly the Insight API.

You only need to use PlaceOrder in very specific scenarios detailed below.

Do I need to use the PlaceOrder mutation?‚Äã

You need to use the PlaceOrder mutation because:

  • you want to support an unsupported platform (supported platform list here)
  • you want to capture orders outside of the platform for instance going headless, building a mobile app, etc.
  • you want to implement a different order capture mechanism on a supported platform (from your CRM, an ERP) leveraging the rating API

To successfully call PlaceOrder, you need to generate rates via the rating API.

HeaderDescription
X-ShipperHQ-Secret-TokenThe secret token that you have generated to use the rating API
X-ShipperHQ-ScopeThe Scope for this ShipperHQ account. If unsure use ‚ÄúLIVE‚Äù (note: the Scope functionality is only available for Magento for now

Example Request‚Äã

Example Request
query placeOrder {
  placeOrder( placeOrderInfo: {
    orderNumber: "1234",      
     totalCharges: 9.85,                    # totalCharges in rating API response
     carrierCode: "shqusps",                # carrierCode in rating API response   
     methodCode: "Priority Mail",           # methodCode in rating API response
     transId: "SHQ_20220201_1625_5144966"   # transactionID from rating API response
  }) {
     transactionId
     responseSummary{
        status
    }
    errors {
      errorCode
      internalErrorMessage
      externalErrorMessage
    }
  }
}
